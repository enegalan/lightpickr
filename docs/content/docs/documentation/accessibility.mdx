## Popover vs inline

- **Popover:** the root `.lp` node gets **<code>role<brown>=</brown><green>"dialog"</green></code>** and **<code>aria-modal<brown>=</brown><green>"true"</green></code>**. It is hidden with <code>display<brown>:</brown> none</code> until opened.
- **Inline:** the calendar is part of the document flow; it is not a modal dialog.

## Grids and labels

- **Day grid:** <code>role<brown>=</brown><green>"grid"</green></code> with **`aria-label`** from **<code><purple>locale.ariaDayGrid</purple></code>**.
- **Month / year grids:** use **`ariaMonthGrid`** / **`ariaYearView`** on their grids.
- **Time:** hours/minutes controls use **`ariaTimeHours`** and **`ariaTimeMinutes`**.
- **Navigation:** prev/next are native **`button`** elements; disabled state sets **`disabled`** and **<code>aria-disabled<brown>=</brown><green>"true"</green></code>** when navigation is out of range.

## Pointer / arrow

The popover pointer element has **<code>aria-hidden<brown>=</brown><green>"true"</green></code>** (decorative).

## Keyboard — calendar

### Change view level

- **<kbd>Alt</kbd> + <kbd>↑</kbd>:** drill **up** (e.g. day → month → year).
- **<kbd>Alt</kbd> + <kbd>↓</kbd>:** drill **down**.

### Day view

- **Arrow keys:** move **`focusDate`** by cell; may change **`viewDate`** when **`moveToOtherMonthsOnSelect`** allows following into adjacent months.
- **<kbd>PgUp</kbd> / <kbd>PgDn</kbd>:** previous/next month (hold **<kbd>Shift</kbd>** for previous/next **year**).
- **<kbd>Home</kbd> / <kbd>End</kbd>:** start/end of the **current week row** in the grid.

### Month and year grids

- **Arrows:** move in a **3-column** grid.
- **<kbd>Home</kbd> / <kbd>End</kbd>:** row start/end within the grid.

### Escape

**<kbd>Esc</kbd>** on `document` closes a **non-inline** popover (<code><purple>hide</purple><gray>()</gray></code>).

## Focus management

- When the popover opens, the library focuses the active cell (<code>tabindex<brown>=</brown><green>"0"</green></code> on the focused day/month/year via **`data-lp-day`**, **`data-lp-month`**, **`data-lp-year`**).
- There is **no** built-in **focus trap**: Tab can move outside the dialog. Use a plugin or app-level focus management if you require trapping for WCAG strict modal behaviour.

## Mobile backdrop

**<code><purple>isMobile</purple></code>** uses a full-screen **`lp-mobile-backdrop`** flex container so the dialog has a predictable touch target layer.

## Customization risks

- **<code><purple>render</purple></code>** option properties can inject arbitrary **HTML**; keep content safe if it includes user data.
- **<code><purple>navTitles</purple></code>** templates use **`innerHTML`**; only use trusted strings or escape input.
