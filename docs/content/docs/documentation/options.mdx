## Mounting & layout

<heading>
### inline
<definition><param-type>boolean</param-type> <brown>=</brown> <red>false</red></definition>
</heading>

Makes the calendar to be permanently visible

<InlineDemo />

<heading>
### isMobile
<definition><param-type>boolean</param-type> <brown>=</brown> <red>false</red></definition>
</heading>

If <red>true</red>, then the calendar will appear as a modal window with slightly enlarged dimensions.

<heading>
### position
<definition><param-type><green>string</green> <brown>|</brown> <blue>function</blue></param-type> <brown>=</brown> <green>"bottom left"</green></definition>
</heading>

If it is a string then the first value is position along the main axis, the second is position along the secondary one. For example, <gray>{'{'}</gray><red>position</red><brown>:</brown> <green>'top right'</green><gray>{'\}'}</gray> — set the calendar position to the top right of the text field.

If **<code><purple>position</purple></code>** is a **function**, it runs when the popover is positioned. The default string placement is skipped—you move and size the panel yourself. Any enter/exit animation is also yours to implement if you want it.

<Callout type="warn" title="Inline and mobile">
  A custom **<code><purple>position</purple></code>** function is **not** used when **<code><purple>inline</purple></code>** or **<code><purple>isMobile</purple></code>** is enabled.
</Callout>

- **`ctx`** — object passed on each layout pass:

| Field          | Type          | Role                                                                                                                               |
|----------------|---------------|------------------------------------------------------------------------------------------------------------------------------------|
| `$datepicker`  | `HTMLElement` | Root `.lp` panel                                                                                                                   |
| `$target`      | `HTMLElement` | Field element from the constructor                                                                                                 |
| `$anchor`      | `HTMLElement` | Placement reference (same as **<code><purple>anchor</purple></code>** option when set, otherwise the field)                        |
| `$pointer`     | `HTMLElement` | Arrow element                                                                                                                      |
| `isViewChange` | `boolean`     | <red>true</red> when this pass is due to a view/month change, not only the first open after **<green>show</green><gray>()</gray>** |

- **Return value:** omit or return <gray>`undefined`</gray> for no custom teardown. Return a **function** to run when the popover closes; that function receives **<green>hideDone</green><gray>()</gray>**. After your teardown, you **must** call **<green>hideDone</green><gray>()</gray>** so the library can finish hiding and detach listeners.

Example of manual positioning:

<ManualPositionDemo />

```js
new Lightpickr('#my-input', {
  autoClose: true,
  position({ $datepicker, $target, $pointer }) {
    const coords = $target.getBoundingClientRect();
    const dpHeight = $datepicker.clientHeight;
    const dpWidth = $datepicker.clientWidth;

    const top = coords.top + coords.height / 2 - dpHeight / 2;
    const left = coords.left + coords.width / 2 - dpWidth / 2;

    $datepicker.style.position = 'fixed';
    $datepicker.style.left = `${left}px`;
    $datepicker.style.top = `${top}px`;

    $pointer.style.display = 'none';
  },
});
```

<heading>
### anchor
<definition><param-type>HTMLElement <brown>|</brown> <green>string</green> <brown>|</brown> null</param-type> <brown>=</brown> <gray>null</gray></definition>
</heading>

Optional node the popover is **aligned to**. Pass an **`HTMLElement`** or **selector** to use a wrapper, icon, etc. If that node is not the constructor **<purple>$target</purple>**, **<purple>showEvent</purple>** is wired to it as well. Unmatched selectors fall back to the default reference.

<definition><param-type>boolean</param-type> <brown>=</brown> <red>true</red></definition>
</heading>

When <red>true</red>, the popover hides after a complete selection: single date, or the second endpoint of a **range**. In **multiple** mode it does not close after each day. Set to <red>false</red> to keep the calendar open after selection.

<definition><param-type><green>string</green></param-type> <brown>=</brown> <green>", "</green></definition>
</heading>

Separator between dates in text field. It is used in the multiple date selection mode and in the range mode.

## Locale & formatting

<heading>
### locale
<definition><param-type>object <brown>|</brown> <green>string</green></param-type> <brown>=</brown> <green>"default"</green></definition>
</heading>

Language of the calendar. Available locales are in <code>lightpickr<brown>/</brown>locale<brown>/</brown></code>.

Supports **full object** from <code>lightpickr<brown>/</brown>locale<brown>/</brown>*</code> or a **partial** object. See [Localization](/docs/documentation/localization).

<heading>
### firstDay
<definition><param-type>number</param-type> <brown>=</brown> <red>1</red></definition>
</heading>

The index of the day from which the week begins. Possible values are from 0 (Sunday) to 6 (Saturday).

<heading>
### weekends
<definition><param-type>number<gray>[]</gray></param-type> <brown>=</brown> <gray>[</gray><red>6</red><gray>, </gray><red>0</red><gray>]</gray></definition>
</heading>

Indexes of the days that will be considered a weekend. The `.-weekend-` class will be added to them. By default, this is Saturday and Sunday.

<heading>
### format
<definition><param-type><green>string</green> <brown>|</brown> <blue>function</blue></param-type> <brown>=</brown> <green>"YYYY-MM-DD"</green></definition>
</heading>

Date format. By default, the **<code><green>string</green></code>** pattern is taken from the current locale; a value passed here overrides that. You can also pass a **<code><purple>function</purple></code>** to build the field value yourself (see below).

**Available tokens** when **<code><purple>format</purple></code>** is a string (longer sequences match first; parsing is left-to-right):

| Token           | Meaning                                                                                                                                            |
|-----------------|----------------------------------------------------------------------------------------------------------------------------------------------------|
| `T`             | Selected instant as milliseconds (Unix timestamp)                                                                                                  |
| `E`             | Short weekday name from locale **<code><purple>weekdaysShort</purple></code>**                                                                     |
| `EEEE`          | Full weekday name from locale **<code><purple>weekdaysLong</purple></code>**                                                                       |
| `d`             | Day of month, 1–31                                                                                                                                 |
| `dd` / `DD`     | Day of month with leading zero                                                                                                                     |
| `M`             | Month number 1–12                                                                                                                                  |
| `MM`            | Month number with leading zero (01–12)                                                                                                             |
| `MMM`           | Short month name from locale (same source as **<code><purple>monthsField</purple></code>**, usually **<code><purple>monthsShort</purple></code>**) |
| `MMMM`          | Full month name from locale **<code><purple>monthsLong</purple></code>**                                                                           |
| `yy`            | Two-digit year                                                                                                                                     |
| `yyyy` / `YYYY` | Four-digit year                                                                                                                                    |
| `yyyy1`         | First year of the current year block                                                                                                               |
| `yyyy2`         | Last year of that block                                                                                                                            |
| `HH`            | Hours 00–23                                                                                                                                        |
| `mm`            | Minutes 00–59                                                                                                                                      |

There is no escape for literals inside a string pattern; avoid embedding these letters in fixed text, or use a function **<code><purple>format</purple></code>** instead.

**Function <code><purple>format</purple></code>:** should return a string. For a single selection, the argument is one <blue>Date</blue>. If **multi-date** selection is enabled, a <code><blue>Date</blue><gray>[]</gray></code> of all selected dates is passed so you can build one string for the whole field. With <code><purple>range</purple></code>, each segment is formatted with <gray>[</gray>startDate<gray>,</gray> endDate<gray>]</gray>; multiple ranges are joined with <code><green>multipleSeparator</green></code>.

<FormatFunctionDemo />

```js
new Lightpickr('#input', {
  format(date) {
    return date.toLocaleString('ja', {
      year: 'numeric',
      day: '2-digit',
      month: 'long',
    });
  },
});
```

<Callout type="tip" title="Multiple selection">
  With <green>multiple</green><brown>:</brown> <red>true</red>, implement against an array, for example <code><purple>const</purple> <red>dates</red> <brown>=</brown> <blue>Array.isArray</blue><gray>(</gray>date<gray>)</gray> <brown>?</brown> date <brown>:</brown> <gray>[</gray>date<gray>]</gray></code> before formatting.
</Callout>

<heading>
### monthsField
<definition><param-type><green>string</green></param-type> <brown>=</brown> <green>"monthsShort"</green></definition>
</heading>

Locale key used for **short** month names in the month grid and for the **`MMM`** format token. Long names always come from **<purple>monthsLong</purple>**.

## Custom DOM

<heading>
### render
<definition><param-type><purple>Partial</purple>&lt;&#123; container, header, time, footer, cell &#125;&gt;</param-type></definition>
</heading>

Every hook uses the same <code><purple>ctx</purple></code> shape (table below). Each is a function <code><gray>(</gray>ctx<gray>)</gray> <brown>=></brown> <blue>HTMLElement</blue> <brown>|</brown> <gray>null</gray> <brown>|</brown> <gray>undefined</gray></code>: return an element to **replace** that part of the UI, or return <gray>null</gray>/<gray>undefined</gray> (or omit the hook key) so the library keeps its default markup.

The same <code><purple>ctx</purple></code> fields apply to every hook:

| Field                         | Type      | Role                                                                                                                 |
|-------------------------------|-----------|----------------------------------------------------------------------------------------------------------------------|
| `date`                        | `number`  | Start-of-day timestamp for the context day (usually the month's anchor, i.e. <code><purple>viewDate</purple></code>) |
| `viewDate`                    | `number`  | Current view anchor (start of that day)                                                                              |
| `isSelected`                  | `boolean` | Day is selected (or part of a range)                                                                                 |
| `isDisabled`                  | `boolean` | Day is not selectable                                                                                                |
| `isToday`                     | `boolean` | Day is "today"                                                                                                       |
| `isInRange`                   | `boolean` | Inside a completed range                                                                                             |
| `isRangeStart` / `isRangeEnd` | `boolean` | Range endpoints                                                                                                      |
| `isFocused`                   | `boolean` | Keyboard focus is on this day                                                                                        |
| `isOutside`                   | `boolean` | For day cells: outside the visible month (<code><purple>showOtherMonths</purple></code>)                             |
| `isWeekend`                   | `boolean` | Weekday index is in <code><purple>weekends</purple></code>                                                           |
| `state`                       | `object`  | Read-only snapshot of public options/state (locale, selection, <code><purple>currentView</purple></code>, etc.)      |
| `instance`                    | `object`  | The Lightpickr instance                                                                                              |

**Hook roles**

- **`container`** — If the function returns an element, it is appended under the root <code><purple>.lp</purple></code> panel and the active view + footer mount **inside** that node. If you define the hook but return <gray>null</gray>, behavior matches having no hook.
- **`header`** — Return a node to place inside the header wrapper for **day**, **month**, and **year** views (same <code><purple>ctx</purple></code>; use <code><purple>state.currentView</purple></code> to branch). Return <gray>null</gray> for the default prev / title / next bar.
- **`time`** — When time is enabled (<code><purple>enableTime</purple></code> or <code><purple>onlyTime</purple></code>), return the full inner content for the time panel. Omit or return <gray>null</gray> for the default time UI.
- **`footer`** — Return a node to use as the footer body. When this hook returns an element, the <code><purple>buttons</purple></code> footer is not used for that render. Omit or return <gray>null</gray> to use the default footer from <code><purple>buttons</purple></code>.
- **`cell`** — Return a root element for that calendar cell (typically a <code><purple>button</purple></code>), or <gray>null</gray>/<gray>undefined</gray> to use the built-in cell. Mirror the default structure from the stylesheet (<code><purple>options.classes</purple></code>) and <code><purple>options.attributes</purple></code> so clicks, keyboard, and range styling still work.

```js
new Lightpickr('#my-input', {
  inline: true,
  render: {
    header(ctx) {
      const el = document.createElement('div');
      el.textContent = 'Pick a date';
      return el;
    },
  },
});
```

```js
new Lightpickr('#my-input', {
  render: {
    cell(ctx) {
      const v = ctx.state.currentView;
      if (v !== 'day') return; // Only day view
      if (new Date(ctx.date).getDay() !== 5) return; // Only day 5 of current view month
      const b = document.createElement('button');
      b.type = 'button';
      b.className = 'lp-cell my-friday';
      b.setAttribute('data-lp-day', String(ctx.date));
      b.setAttribute('role', 'gridcell');
      b.setAttribute('tabindex', ctx.isFocused ? '0' : '-1');
      b.setAttribute('aria-selected', ctx.isSelected ? 'true' : 'false');
      b.setAttribute('aria-disabled', ctx.isDisabled ? 'true' : 'false');
      b.textContent = String(new Date(ctx.date).getDate());
      if (ctx.isDisabled) b.disabled = true;
      return b;
    },
  },
});
```
