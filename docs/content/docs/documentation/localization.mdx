There is **no** <code>lang: <green>"es"</green></code> string switch in the constructor. You pass a **<code><purple>locale</purple></code> object**: either the full module from `lightpickr/locale/*` or a **partial** object merged on top of English defaults.

## Importing a bundled locale

```js
import Lightpickr from 'lightpickr';
import 'lightpickr/lightpickr.css';
import es from 'lightpickr/locale/es';

new Lightpickr('#field', {locale: es});
```

## Partial custom locale

```js
new Lightpickr('#field', {
  locale: {
    btnToday: 'Go to today',
    ariaDayGrid: 'Selectable dates',
    monthsShort: ['J', 'F', 'M', 'A', 'M', 'J', 'J', 'A', 'S', 'O', 'N', 'D'],
  },
});
```

You can mix **import + spread** to tweak one string:

```js
import es from 'lightpickr/locale/es';

new Lightpickr('#field', {
  locale: {...es, btnClear: 'Vaciar'},
});
```
